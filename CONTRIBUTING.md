# Contributing Guide

Thanks for contributing to SDTK.

## Workflow
1. Open or link an issue.
2. Keep changes scoped and traceable.
3. Update docs/templates/skills together when behavior changes.
4. Provide validation evidence (commands + results).

## Pull Requests
- Use the PR template in `.github/PULL_REQUEST_TEMPLATE.md`.
- Explain rationale and risk.
- Include rollback strategy for behavior changes.
- Ensure required quality checks pass.

## Commit and Review Expectations
- Prefer small, focused PRs.
- Do not include secrets or local machine artifacts.
- Keep generated content deterministic and reviewable.
